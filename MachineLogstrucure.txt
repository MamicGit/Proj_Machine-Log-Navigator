*************************************
**       Identifyer Log columns    **
*************************************
-> hat Mohamad bereits

    DATE/TIME                       → whitespace (z. B. "2026-02-23 20:11:44,503")
    Severity                        → whitespace vor/nach (z. B. "INFO")
    THREAD                          → eckige Klammern [ ] (z. B. "[metrics-agent]")
    SOURCE/LOGGER                   → whitespace getrennt, oft FQCN direkt nach Thread (z. B. "com.example.MyClass")
    MESSAGE/PREFIX                  → space + hyphen + space " - " (trennt Header von Message)
        KEY=VALUE im Message            → Semikolon + space "; " (oder Komma + space ", ")
        Zusammengesetzte Header‑Items   → Pipe "|" innerhalb der Klammer (z. B. "[SCADA|conn=FD-127]")
        HOST/PID/TID                    → oft in Klammern "(host=host1 pid=1234)" oder als weitere token in [..]
        TRACE/REQUEST IDs               → meist als token im Header oder als "key=value" im Message, getrennt mit whitespace oder "; "
        STACKTRACE                      → newline (mehrzeilig), beginnt direkt nach Message und eingerückt
